git commit - створює новий комміт

git branch - створює нову гілку ( branch -f from to переміщає насильно гілку, вказуємо яку гілку і куди )

git checkout - вибирає активну гілку чи комміт

git merge - обєднує гілки

git rebase - копіює активний комміт у вказаний ( git rebase -i [до куди копыювати] - інтерактивний rebase, який може копіювати декільки комітів одночасно і в різному порядку )

HEAD - активний комміт

git reset ['кількість кроків'] - відміняє зміни і переміщає вказівник гілки на попередній комміт (використовуємо для локального використання)

git revert - створює новий комміт в який заносить зміни які протилежні тим  які були в комміті на основі якого зроблений новий комміт

git cherry-pick <комміт1> <комміт2> <комміт3> ... - копіює вибрані комміти у активну гілку

git tag [name] [commit] - добавляє теги вираному комміту, якщо не вказаний комміт то в активний комміт
